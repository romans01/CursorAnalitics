# Реестр заинтересованных сторон

## Назначение
Полный список всех лиц, групп и организаций, которые могут влиять на проект или подвергаться его влиянию.

## Классификация стейкхолдеров

### Внутренние стейкхолдеры
**Руководство компании**
- **Роль:** CEO, CTO, Директор по развитию
- **Интерес:** ROI, стратегическое развитие, конкурентные преимущества
- **Влияние:** Высокое (принятие ключевых решений, бюджет)
- **Отношение:** Поддерживающее
- **Коммуникация:** Месячные отчеты о прогрессе, презентации результатов

**Бизнес-пользователи**
- **Роль:** Менеджеры отделов, аналитики, операционные сотрудники
- **Интерес:** Удобство работы, автоматизация рутины, новые возможности
- **Влияние:** Среднее (требования, приемка, обратная связь)
- **Отношение:** Заинтересованное, но с опасениями
- **Коммуникация:** Еженедельные демо, воркшопы, обучение

**IT Департамент**
- **Роль:** Системные администраторы, разработчики, архитекторы
- **Интерес:** Техническая совместимость, поддержка, безопасность
- **Влияние:** Высокое (техническая реализация, интеграции)
- **Отношение:** Нейтральное (зависит от сложности)
- **Коммуникация:** Технические ревью, архитектурные сессии

### Внешние стейкхолдеры
**Клиенты компании**
- **Роль:** Конечные потребители услуг/продуктов
- **Интерес:** Качество сервиса, скорость обслуживания
- **Влияние:** Среднее (косвенное через обратную связь)
- **Отношение:** Нейтральное (не знают о проекте)
- **Коммуникация:** Опросы удовлетворенности, фокус-группы

**Партнеры и поставщики**
- **Роль:** Интеграционные партнеры, поставщики данных
- **Интерес:** Совместимость систем, объемы интеграций
- **Влияние:** Среднее (техническая интеграция)
- **Отношение:** Заинтересованное в сотрудничестве
- **Коммуникация:** Технические консультации, тестирование

**Регуляторы**
- **Роль:** Контролирующие органы (ЦБ, Роскомнадзор и др.)
- **Интерес:** Соответствие требованиям, защита данных
- **Влияние:** Высокое (нормативные требования)
- **Отношение:** Нейтральное (соблюдение требований)
- **Коммуникация:** Официальные отчеты, аудиты

## Анализ стейкхолдеров

### Ключевые влиятельные стейкхолдеры
1. **Иванов И.И. (CEO)** - критически важен для успеха проекта
   - Стратегия взаимодействия: регулярные отчеты о ROI и прогрессе
   - Риски: потеря поддержки при задержках или превышении бюджета

2. **Петров П.П. (IT Директор)** - ключевой для технической реализации
   - Стратегия взаимодействия: вовлечение в архитектурные решения
   - Риски: технические блокеры при несогласии с решениями

### Потенциальные противники
1. **Сотрудники отделов** - могут сопротивляться изменениям
   - Митигация: раннее вовлечение, демонстрация пользы, обучение
   - Коммуникация: акцент на том, как проект упростит их работу

### Нейтральные стейкхолдеры
1. **Клиенты** - не знают о проекте, но получат пользу
   - Стратегия: мониторинг удовлетворенности, сбор обратной связи

## Стратегии взаимодействия

### Высокое влияние, высокий интерес (Управлять вплотную)
- CEO, IT Директор
- **Действия:** Регулярные встречи, детальная отчетность, вовлечение в решения

### Высокое влияние, низкий интерес (Поддерживать удовлетворенность)
- Финансовый директор, HR директор  
- **Действия:** Информирование о ключевых вехах, демонстрация ROI

### Низкое влияние, высокий интерес (Информировать)
- Конечные пользователи, техническая поддержка
- **Действия:** Регулярные обновления, сбор обратной связи, обучение

### Низкое влияние, низкий интерес (Мониторить)
- Клиенты, некоторые партнеры
- **Действия:** Периодическое информирование, мониторинг изменений

## План коммуникаций со стейкхолдерами

### Еженедельно
- Статус-колл с командой проекта
- Обновления для ключевых пользователей
- Технические консультации с IT

### Ежемесячно  
- Отчет для руководства (CEO, директора)
- Демо новой функциональности
- Анализ обратной связи от пользователей

### По вехам проекта
- Презентации результатов для всех стейкхолдеров
- Планирование следующих этапов
- Корректировка стратегий взаимодействия

## Управление изменениями в составе стейкхолдеров

### Процесс добавления новых стейкхолдеров
1. Идентификация нового стейкхолдера
2. Анализ интересов и влияния
3. Определение стратегии взаимодействия
4. Обновление реестра и планов коммуникации
5. Уведомление команды проекта

### Мониторинг изменений
- Ежемесячный пересмотр реестра
- Отслеживание изменений в организационной структуре
- Анализ обратной связи и изменений в отношении к проекту
