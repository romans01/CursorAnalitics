## 3. Функциональные требования

### 3.1 Управление пользователями

#### ФТ-001: Регистрация пользователя
**Описание:** Система должна позволять новым пользователям создавать учетные записи.

**Входные данные:**
- Email адрес (обязательно)
- Пароль (обязательно, минимум 8 символов)
- Имя и фамилия (обязательно)
- Телефон (опционально)

**Обработка:**
1. Валидация введенных данных
2. Проверка уникальности email
3. Хеширование пароля
4. Создание записи в БД
5. Отправка письма подтверждения

**Выходные данные:**
- Подтверждение успешной регистрации
- Email с ссылкой активации

**Бизнес-правила:**
- BR-001: Email должен быть уникальным в системе
- BR-002: Пароль должен содержать минимум 8 символов
- BR-003: Активация аккаунта обязательна в течение 24 часов

**Критерии приемки:**
- [ ] Пользователь может зарегистрироваться с валидными данными
- [ ] Система отклоняет дублирующие email адреса
- [ ] Отправляется письмо подтверждения
- [ ] Неактивированные аккаунты удаляются через 24 часа

#### ФТ-002: Аутентификация пользователя
**Описание:** Система должна проверять подлинность пользователей при входе.

**Входные данные:**
- Email адрес
- Пароль

**Обработка:**
1. Проверка существования пользователя
2. Верификация пароля
3. Генерация JWT токена
4. Логирование входа

**Выходные данные:**
- JWT токен доступа
- Информация о пользователе
- Время истечения сессии

**Исключительные ситуации:**
- Неверные учетные данные → Ошибка "Неправильный email или пароль"
- Заблокированный аккаунт → Ошибка "Аккаунт временно заблокирован"
- Неактивированный аккаунт → Ошибка "Необходимо активировать аккаунт"

### 3.2 Управление данными

#### ФТ-003: Создание записи
**Описание:** Пользователи должны иметь возможность создавать новые записи в системе.

**Предусловия:**
- Пользователь аутентифицирован
- У пользователя есть права на создание записей

**Основной поток:**
1. Пользователь выбирает "Создать новую запись"
2. Система отображает форму создания
3. Пользователь заполняет обязательные поля
4. Пользователь нажимает "Сохранить"
5. Система валидирует данные
6. Система сохраняет запись в БД
7. Система отображает подтверждение

**Альтернативные потоки:**
- 5a. Данные не прошли валидацию
  - 5a1. Система показывает ошибки валидации
  - 5a2. Пользователь исправляет данные
  - 5a3. Возврат к шагу 4

**Постусловия:**
- Новая запись создана в системе
- Пользователь уведомлен об успешном создании

## 4. Нефункциональные требования

### 4.1 Производительность
**НФТ-001: Время отклика**
- Время отклика веб-страниц: не более 2 секунд при нормальной нагрузке
- Время выполнения поисковых запросов: не более 5 секунд
- Время генерации отчетов: не более 30 секунд

**НФТ-002: Пропускная способность**
- Система должна поддерживать до 1000 одновременных пользователей
- Обработка до 10000 транзакций в час
- Пиковая нагрузка: 150% от нормальной в течение 1 часа

**НФТ-003: Масштабируемость**
- Возможность горизонтального масштабирования
- Поддержка до 100000 пользователей
- Рост данных: до 1TB в год

### 4.2 Надежность
**НФТ-004: Доступность**
- Время работы системы: 99.5% (не более 36 часов простоя в год)
- Плановые технические работы: не более 4 часов в месяц
- Время восстановления после сбоя: не более 4 часов

**НФТ-005: Отказоустойчивость**
- Резервное копирование данных каждые 24 часа
- Репликация критичных данных в реальном времени
- Автоматическое переключение на резервные системы

### 4.3 Безопасность
**НФТ-006: Аутентификация и авторизация**
- Многофакторная аутентификация для администраторов
- Ролевая модель доступа (RBAC)
- Автоматическая блокировка после 5 неудачных попыток входа

**НФТ-007: Защита данных**
- Шифрование персональных данных в БД (AES-256)
- Использование HTTPS для всех соединений
- Логирование всех действий пользователей

**НФТ-008: Соответствие требованиям**
- Соответствие GDPR для обработки персональных данных
- Соответствие ISO 27001 для информационной безопасности
- Регулярные аудиты безопасности

### 4.4 Удобство использования
**НФТ-009: Пользовательский интерфейс**
- Адаптивный дизайн для устройств с разрешением от 320px
- Поддержка основных браузеров (Chrome, Firefox, Safari, Edge)
- Время обучения новых пользователей: не более 2 часов

**НФТ-010: Доступность**
- Соответствие стандартам WCAG 2.1 уровня AA
- Поддержка программ чтения с экрана
- Навигация с клавиатуры

### 4.5 Совместимость
**НФТ-011: Интеграции**
- REST API для интеграции с внешними системами
- Поддержка форматов JSON, XML для обмена данными
- Совместимость с существующими системами компании
