# Глоссарий проекта

## Назначение
Единый словарь терминов и определений, используемых в проекте. 
Обеспечивает общее понимание ключевых концепций всеми участниками команды.

## Принципы ведения глоссария
- ✅ Используем простые и понятные определения
- ✅ Избегаем циклических ссылок в определениях
- ✅ Указываем синонимы и сокращения
- ✅ Регулярно обновляем при появлении новых терминов

## Бизнес-термины

### А
**Аналитика (Analytics)**
Процесс систематического анализа данных для извлечения полезной информации и поддержки принятия решений.
*Синонимы:* Data Analytics, Анализ данных

**API (Application Programming Interface)**
Набор протоколов и инструментов для создания программных приложений, определяющий способы взаимодействия компонентов.
*Пример:* REST API для интеграции с CRM системой

### Б
**Бизнес-процесс (Business Process)**
Последовательность связанных действий или задач, выполняемых для достижения определенного бизнес-результата.
*Связанные термины:* Workflow, Process Map

**Бизнес-правило (Business Rule)**
Формальное утверждение, определяющее или ограничивающее некоторый аспект бизнеса.
*Пример:* "Скидка не может превышать 50% от стоимости товара"

### В
**Витрина данных (Data Mart)**
Специализированное подмножество хранилища данных, ориентированное на конкретную бизнес-область или группу пользователей.
*Связанные термины:* Data Warehouse, OLAP

### Д
**Дашборд (Dashboard)**
Интерактивный интерфейс, отображающий ключевые показатели эффективности (KPI) в виде графиков, диаграмм и таблиц.
*Синонимы:* Панель мониторинга, Информационная панель

**Данные (Data)**
Факты, цифры и статистика, собранные для анализа или справочных целей.
*Типы:* Структурированные, Неструктурированные, Полуструктурированные

### И
**Интеграция (Integration)**
Процесс связывания различных систем и приложений для обеспечения их совместной работы.
*Типы:* Синхронная, Асинхронная, Batch, Real-time

### К
**KPI (Key Performance Indicator)**
Ключевой показатель эффективности - измеримое значение, демонстрирующее насколько эффективно достигаются основные бизнес-цели.
*Примеры:* Конверсия, ROI, Customer Lifetime Value

### М
**Метрика (Metric)**
Количественная мера, используемая для отслеживания и оценки статуса конкретного бизнес-процесса.
*Отличие от KPI:* Метрика - это измерение, KPI - это ключевая метрика для бизнеса

### О
**OLAP (Online Analytical Processing)**
Технология обработки данных, позволяющая аналитикам быстро получать ответы на многомерные аналитические запросы.
*Операции:* Drill-down, Roll-up, Slice, Dice

### П
**Пользовательская история (User Story)**
Краткое описание функциональности с точки зрения пользователя в формате: "Как [роль], я хочу [действие], чтобы [цель]".
*Компоненты:* Роль, Действие, Ценность, Критерии приемки

### Р
**Репозиторий (Repository)**
Централизованное место хранения и управления данными, кодом или другими артефактами проекта.
*Типы:* Git Repository, Data Repository, Artifact Repository

### С
**Система (System)**
Организованная совокупность взаимосвязанных компонентов, работающих вместе для достижения общей цели.
*Примеры:* CRM система, ERP система, BI система

**Стейкхолдер (Stakeholder)**
Лицо, группа или организация, которые могут влиять на проект или подвергаться его влиянию.
*Синонимы:* Заинтересованная сторона, Участник проекта

### Т
**Трассируемость (Traceability)**
Способность отследить связи между требованиями, дизайном, кодом и тестами на протяжении всего жизненного цикла проекта.
*Типы:* Прямая трассируемость, Обратная трассируемость

### Э
**Эпик (Epic)**
Крупная пользовательская история, которая слишком большая для реализации в одном спринте и требует разбиения на более мелкие истории.
*Связанные термины:* User Story, Feature, Theme

## Технические термины

### A
**API Gateway**
Сервис, который выступает в качестве единой точки входа для всех клиентских запросов к микросервисам.
*Функции:* Маршрутизация, Аутентификация, Rate Limiting

### E
**ETL (Extract, Transform, Load)**
Процесс извлечения данных из источников, их преобразования и загрузки в целевую систему.
*Альтернатива:* ELT (Extract, Load, Transform)

### R
**REST (Representational State Transfer)**
Архитектурный стиль для создания веб-сервисов, основанный на стандартных HTTP методах.
*Принципы:* Stateless, Cacheable, Uniform Interface

## Роли и ответственности

### Аналитик данных (Data Analyst)**
Специалист, который собирает, обрабатывает и анализирует данные для поддержки бизнес-решений.
*Ответственности:* Создание отчетов, анализ трендов, подготовка рекомендаций

**Бизнес-аналитик (Business Analyst)**
Специалист, который анализирует бизнес-процессы и требования для определения потребностей в изменениях.
*Ответственности:* Сбор требований, анализ процессов, документирование

**Product Owner**
Роль в Scrum, ответственная за определение и приоритизацию требований к продукту.
*Ответственности:* Управление Product Backlog, определение критериев приемки
